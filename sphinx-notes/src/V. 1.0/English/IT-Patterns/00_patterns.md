# Patterns

Patterns in computer science are recurring, generalizable solutions to common problems that arise in software design and development. They are not directly reusable code, but conceptual schemas that describe a solution structure applicable across different contexts.

The concept was first formalized in the field of architecture by Christopher Alexander and later adopted in computer science by the so-called **Gang of Four (GoF)**, Gamma, Helm, Johnson, and Vlissides, with the publication of *Design Patterns: Elements of Reusable Object-Oriented Software* (1994), which catalogued 23 fundamental patterns.

Patterns are generally classified across multiple levels of abstraction:

- **Architectural patterns**, define the high-level organization of a system (e.g. MVC, Event-Driven, Layered)
- **Design patterns**, concern the structure and relationships between components at the code level (e.g. Observer, Strategy, Template Method)
- **Idioms**, low-level solutions specific to a particular language or platform

A pattern is not a rigid rule but a shared vocabulary among developers: recognizing and naming a pattern allows complex architectural solutions to be communicated precisely and efficiently, reducing ambiguity in system design.
